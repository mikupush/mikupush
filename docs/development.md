# Development

## Prerequisites

- [Node.js](https://nodejs.org/en) 22 or later
- [Rust](https://rust-lang.org/) 1.93 or later
- [Install required packages by Tauri](https://v2.tauri.app/start/prerequisites/#system-dependencies)

### Windows

- [Visual Studio](https://visualstudio.microsoft.com/) 2022 or later
    - 👉 [Required workloads](../helpers/windows-helpers/README.md)

### MacOS

- XCode 26 or later (you can try with older version, but we stared with XCode 26)

## Building

You can build the release binary following the [Building from source](https://mikupush.io/docs/client/building-from-source/) docs.

## Local development

You can use HMR on the webview and restart the program when changes are applied running `tauri dev`

```sh
npm run tauri dev
```

On macOS, you should build first the share extension or removing the referente to it on `src-tauri/tauri.config.json`.

```sh
bash scripts/build-macos-helpers.sh
```

Or opening the project `helpers/macos-helpers` with XCode and building from it.

### Database

We use SQLite as database with [Diesel](https://diesel.rs/) for save user servers and file uploads.

> ℹ️ [Install Diesel CLI](https://diesel.rs/guides/getting-started#installing-diesel-cli) before manage the database schema.

For modify the database schema or update or add new rows to existing tables, we should create a new migration.

```sh
diesel migration generate <migration name in snake-case>
```

Then apply the migrations to `schema.rs`

> ℹ️ Remember not modify `schema.rs` because it is autogenerated by Diesel

```sh
diesel migration run
```

After that you must update the corresponding struct if you modified an existing table.
